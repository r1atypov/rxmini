# rxmini

1. Цель работы
Цель проекта — реализовать упрощённую версию библиотеки RxJava, включающую:

реактивные потоки данных;

обработку ошибок;

асинхронное выполнение через Schedulers;

базовые операторы (map, filter, flatMap);

управление подписками (Disposable);

тестирование всех ключевых компонентов.

2. Архитектура системы
2.1. Основные сущности
Observable
Представляет поток данных. Хранит функцию OnSubscribe, которая вызывается при подписке.

Observer
Получатель данных. Имеет методы:

onNext(T item)

onError(Throwable t)

onComplete()

Disposable
Позволяет отменить подписку и прекратить получение событий.

Scheduler
Абстракция над пулом потоков. Позволяет выполнять задачи асинхронно.

3. Реализованные операторы
map
Преобразует элементы потока.

filter
Отбрасывает элементы, не удовлетворяющие условию.

flatMap
Преобразует элемент в новый Observable и объединяет результаты.

subscribeOn
Определяет поток, в котором выполняется подписка.

observeOn
Определяет поток, в котором вызываются методы Observer.

4. Реализация Schedulers
IOThreadScheduler
Использует CachedThreadPool.
Подходит для I/O‑операций, сетевых запросов, работы с файлами.

ComputationScheduler
Использует FixedThreadPool с количеством потоков = числу ядер CPU.
Подходит для CPU‑интенсивных задач.

SingleThreadScheduler
Использует один поток.
Подходит для последовательных операций, логирования, UI‑подобных сценариев.

5. Обработка ошибок
Ошибки передаются через onError().
После вызова onError() поток завершается, дальнейшие события игнорируются.

6. Управление подписками
Disposable позволяет:

прекратить получение событий;

предотвратить вызовы onNext() после отмены.

7. Тестирование
Проведены тесты:

корректность map, filter, flatMap;

корректность обработки ошибок;

работа Disposable;

работа subscribeOn и observeOn в многопоточной среде;

корректность Schedulers.
